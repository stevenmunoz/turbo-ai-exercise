# Brand Voice Rules & Copy Catalog

## Voice Pillars

Define your project's voice pillars here. Common examples:

- **Friendly** — Speak like a helpful friend, not a corporation
- **Casual** — Use contractions, conversational language
- **Empathetic** — Acknowledge the user's situation before offering solutions

## Tone Rules

1. **Contractions always** — "we'll", "you're", "don't", "it's", "didn't"
2. **"You/your" language** — speak directly to the reader
3. **Empathy first** — acknowledge the struggle before offering solutions
4. **Active verbs** — prefer action-oriented language
5. **Parenthetical warmth** — add personality with asides like "(no worries)", "(we've got you)"
6. **No jargon** — if a non-technical person wouldn't understand it, rewrite it
7. **Keep it brief** — respect their time, say it simply
8. **Never patronizing** — warm and helpful, not childish

## Copy Patterns By Category

### Success Messages
Structure: Short celebratory phrase, warm and specific.

```
"Done!"
"All set!"
"Saved!"
"Changes saved successfully."
```

### Processing/Loading States
Structure: Playful action description of what's happening.

```
"Working on it..."
"Almost there..."
"Saving your changes..."
"Just a moment..."
```

### Warning/Confirmation Messages
Structure: Casual question + reassurance + clear actions.

```
Title: "Are you sure?"
Body: "This action can't be undone. Want to go ahead?"
Actions: "Yes, continue" / "Cancel"
```

### Error Messages
Structure: Friendly acknowledgment + what went wrong + what to do.

```
"Something went wrong. Please try again."
"We couldn't connect. Check your internet and try again."
```

Avoid: generic "Error" titles. Use context-specific titles.

### Empty States
Structure: Encouraging nudge + what happens when they take action.

```
"Nothing here yet — let's change that!"
"Get started by adding your first item."
```

### Button Labels
Structure: Active, casual, clear intent.

| Formal | Friendly |
|--------|----------|
| Get Started | Let's Go |
| Submit | Save / Send / Done |
| Continue | Next |
| Retry | Try Again |
| Dismiss | Skip |

### Form Labels & Placeholders
Structure: Conversational question or simple label. Placeholders give examples.

```
Label: "What's your name?"
Placeholder: "Enter your name"

Label: "Email address"
Placeholder: "you@example.com"
```

Optional fields: always suffix with "(optional)"

### Onboarding / First-Time Copy
Structure: Welcoming, low-pressure, action-oriented.

```
"Welcome! Let's get you set up."
"Tell us a bit about yourself"
"Ready to get started?"
```

### Reassurance Phrases (use liberally)
```
"No worries"
"You can always change this later"
"We've got you covered"
"Don't worry, it's safe"
```

## Multilingual Translation Rules

If your project supports multiple languages, add language-specific rules below.

### General Translation Guidelines

1. **Maintain tone** — Translations should feel natural in the target language, not literal
2. **Respect punctuation** — Follow the target language's punctuation conventions
3. **Accents matter** — Always include proper diacritical marks
4. **Cultural adaptation** — Adjust idioms and cultural references for the target audience
